## Observations

1. **LLM fallback works cleanly.** No EURI_API_KEY in env -> template copy generator engaged, JSON shape honoured.
2. **Phone validation normalises** incoming `+919876543210` to E.164 unchanged; would also fix `919876543210` -> `+919876543210`.
3. **Queue is idempotent-friendly.** Each task has a unique `task_id`; re-running the pipeline appends rather than replaces, which is the right behaviour for onboarding (re-runs imply re-engagement).
4. **Secret masking active.** No keys printed even when they would be (log grep clean).
5. **Subprocess orchestration adds ~600ms per step** — acceptable for this throughput class; can be flattened to in-process calls if needed later.
